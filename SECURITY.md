# Security Policy

## Scope

This repository contains a research prototype built for an MBA project on AI and Digital
Transformation in International Tax. It is **not a production system**. The guidance below
applies to anyone running or extending the project.

---

## Legal Disclaimer

The output of this agent is **not legal advice**. Analysis produced by the WHT agent:

- Is generated by a large language model and may contain errors.
- Is based on treaty data that has not been fully verified against official treaty PDFs
  (`verified: false` flag in `data/treaties.json`).
- Does not substitute for review by a qualified Polish tax professional.
- Should not be relied upon for actual withholding tax decisions without independent
  verification.

This software is provided for informational and research purposes only.

---

## API Keys

- API keys (`OPENAI_API_KEY`, `GEMINI_API_KEY`) are loaded from `.env` at runtime.
- `.env` is gitignored — **never commit it**.
- Use `.env.example` (committed, no real values) as the configuration template.
- Do not embed keys in source files, test fixtures, or JSON data files.
- Rotate any key immediately if it is accidentally committed.

---

## Data Handling

- **No PII should be entered into the system.** Entity names used in analysis
  (e.g. "Orange S.A.", "Alpine Holdings") are publicly known legal entities.
- The `data/registry.json` file (generated at runtime) is gitignored. It contains
  analysis outputs — treat it as internal data, not for public sharing.
- The `reports/` directory (generated at runtime) is gitignored for the same reason.
- DDQ documents (`data/ddqs/`) may contain sensitive business information.
  The included `orange_sa_ddq.txt` is a synthetic sample only.

---

## Access Control

The Express web server (`npm start`) has no authentication. Port 3000 is intended for
local development only. **Do not expose port 3000 publicly** — any client that can reach
it can trigger an OpenAI API call at your expense. Rate limiting and authentication are
planned for Phase 22 (Production Hardening).

---

## Dependency Hygiene

- Run `npm audit` before merging any PR that changes `package.json` or `package-lock.json`.
- Review and update dependencies monthly. Pin major versions in `package.json` to avoid
  unexpected breaking changes.
- The Python DDQ service (`python/`) has its own dependencies (`requirements.txt`).
  Apply the same audit discipline: `pip-audit` or manual review before updating.

---

## Reporting Issues

For general bugs and feature requests, open a GitHub issue:
[github.com/fmochnacz-roul-duke/tax-agent-ai/issues](https://github.com/fmochnacz-roul-duke/tax-agent-ai/issues)

For sensitive security concerns (e.g. a vulnerability that could expose API keys or
allow unauthorized analysis), **do not open a public issue**. Contact the author
directly via GitHub ([@fmochnacz-roul-duke](https://github.com/fmochnacz-roul-duke))
with the label `security` in your message subject.
